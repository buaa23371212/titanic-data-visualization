# 泰坦尼克号数据可视化项目 - 技术架构文档

## 架构概述
本项目采用现代前端技术栈，构建一个响应式、交互式的数据可视化应用。架构设计遵循模块化、组件化的原则，确保代码的可维护性和可扩展性。

## 技术栈选择

### 前端框架
- **React 18**: 主流前端框架，提供优秀的组件化开发体验
- **TypeScript**: 类型安全的JavaScript，提高代码质量和开发效率
- **Vite**: 快速的构建工具，提供优秀的开发体验

### 样式和UI
- **Tailwind CSS**: 实用优先的CSS框架，快速构建现代UI
- **CSS Modules**: 组件级样式隔离
- **Responsive Design**: 移动端优先的响应式设计

### 数据可视化
- **D3.js**: 强大的底层数据可视化库，提供数据绑定、比例尺、坐标轴等核心功能
- **React + SVG**: 使用React组件结合SVG元素手动实现可视化效果
- **Canvas API**: 用于高性能的自定义绘图和复杂可视化

### 数据处理
- **Python**: 数据预处理和分析
- **Pandas**: 数据操作和分析
- **NumPy**: 数值计算

### 开发工具
- **ESLint**: 代码质量检查
- **Prettier**: 代码格式化
- **Git**: 版本控制
- **GitHub Actions**: CI/CD自动化

### 2. 可视化组件模块

#### 图表组件分类
1. **基础统计图表**
   - 柱状图、饼图、折线图
   - 使用D3.js + React手动实现

2. **高级交互图表**
   - 散点图、热力图、桑基图
   - 使用D3.js + SVG/Canvas手动实现

3. **复合图表**
   - 仪表盘、多图表联动
   - 组合多个基础图表

#### Observable Framework 组件架构
- **响应式数据流**: 基于Observable的数据流管理
- **组件系统**: 使用Observable Framework的组件架构组织应用
- **交互模式**: 实现数据驱动的交互式组件
- **嵌入式组件**: 创建可复用的数据可视化组件

### 3. 状态管理设计

#### 应用状态结构
```typescript
interface AppState {
  // 数据状态
  passengers: Passenger[];
  loading: boolean;
  
  // 筛选状态
```

#### 状态管理策略
- 使用React Context + useReducer管理全局状态
