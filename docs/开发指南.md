# 泰坦尼克号数据可视化项目 - 开发指南

## 开发环境搭建

### 前置要求
- Node.js 16+ 
- Python 3.8+
- Git
- 代码编辑器 (推荐 VS Code)

### 环境配置步骤

#### 1. 克隆项目
```bash
git clone <项目仓库地址>
cd titanic-data-visualization
```

#### 2. 安装前端依赖
```bash
npm install
# 或使用 yarn
yarn install
```

#### 3. 安装Python依赖
```bash
cd scripts
pip install -r requirements.txt
```

#### 4. 数据预处理
```bash
# 根目录下运行数据预处理脚本
python scripts/data_preprocessing.py
```

#### 5. 启动开发服务器
```bash
npm run dev
# 或使用 yarn
yarn dev
```

### 作业注意事项

#### 重要限制条件
**⚠️ 作业要求**: 不能直接调用 Observable 的 Plot 等直接生成可视化的模块。

这意味着：
- 不能使用 `@observablehq/plot` 库中的 `Plot.plot()`、`Plot.barY()` 等直接生成图表的方法
- 需要基于 D3.js 等底层可视化库手动实现可视化效果
- 可以使用 D3.js 的数据处理功能，但不能使用其高级图表生成器
- 鼓励使用原生 SVG、Canvas 或基础的 DOM 操作来实现可视化

#### 允许使用的 Observable 相关技术
虽然不能直接使用 Plot 生成图表，但可以使用的 Observable 相关技术包括：
- **Observable Framework 的组件系统**: 用于构建数据应用的基础架构
- **Observable 的数据流和响应式编程模式**: 用于管理数据状态和交互
- **Observable 的嵌入式组件 API**: 用于创建可复用的数据组件
- **Observable 的输入控件和交互模式**: 用于构建用户界面

#### 允许使用的其他技术栈
- **D3.js 基础功能**: 选择集、数据绑定、比例尺、坐标轴等
- **React + SVG**: 使用 React 组件结合 SVG 元素手动绘制图表
- **Canvas API**: 使用 Canvas 进行自定义绘图
- **CSS/HTML**: 基础的 HTML 元素和 CSS 样式

### 最佳实践建议

#### 数据可视化设计原则
1. **明确目标**: 每个图表都应该有明确的沟通目标
2. **简化设计**: 避免过度装饰，突出数据本身
3. **交互性**: 提供适当的交互功能增强用户体验
4. **响应式**: 确保图表在不同设备上都能良好显示
